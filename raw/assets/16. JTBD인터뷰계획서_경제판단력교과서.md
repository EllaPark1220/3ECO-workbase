## 1. 인터뷰 개요 및 목적

### 1.1 왜 지금 인터뷰인가 — 1차 시장 분석 요약

지금까지 수행된 분석을 통해 본 프로젝트의 시장 위치가 정량적으로 명확해졌습니다.

**시장 규모 (SAM 심화분석)**
- Theoretical SAM: 약 1,500~1,800만 명
- Strict SAM: 약 300~450만 명
- Early SAM (MVP 대상): 약 100~200만 명
- 1년차 SOM (완주 학습자): 300~1,000명

**경쟁 구조 (5 Forces)**
- 대체재 위협 매우 강함(AI·생성형 모델이 개별 질문 입구를 빠르게 점유)
- 기존 경쟁자 분열 — 『후킹·유료 전환』 축에 몰려 있어 『정확·무료·체계』 축이 구조적 공백으로 존재

**가장 시급한 타깃 (Market Segment Map · Q1 심화세분화)**
- Q1 각성 초심자 사분면. 그중 Q1-A(청년 각성형)가 최우선.
- 시간 창이 2~3년 단위로 닫히는 AI 침투 대상.

**기회 강도 (AOS × DOS 매트릭스)**
- Q1(혁신기회) 7개 Pain: P1 체계감, P5 깊이공백, P14 학생경로, P4 신뢰기준, P7 판단기준, P13 수업부담, P16 세션단편
- 이 7개 Pain이 MVP 집중 영역

### 1.2 이 분석이 해결하지 못하는 질문

위 분석은 **합성 프로필과 공식 통계**에 기반합니다. 이는 몇 가지 핵심 질문에 답할 수 없습니다.

1. **Pain의 실제 언어** — 박지훈·이수민이 실제로 "체계감 부재"를 어떤 표현으로 말하는가?
2. **전환 사건(Switch Event)** — 학습을 결심한 순간 실제로 무엇이 있었는가? 수치 통계는 시점을 알려주지 않음.
3. **대체재와의 실제 비교** — AI·유튜브·책을 왜 "고용"하고 왜 "해고"하는가? 만족도 점수가 아닌 **교체의 순간**이 필요.
4. **가설 AOS·DOS의 실측 보정** — Importance 5, Satisfaction 1이라는 추정이 실제 인터뷰 리커트 응답과 얼마나 일치하는가?
5. **설계 원칙의 수용성** — 원칙 5(1편=1교안=1장)가 정해민(Q1-B)에게 실제로 장벽인가, 수용 가능한가?

### 1.3 JTBD 인터뷰의 필수성

업로드된 인터뷰 가이드 카드의 핵심 원칙을 본 프로젝트에 맞춰 정리하면:

> **JTBD는 사용자가 긍정적인 변화(progress)를 이루기 위해 어떤 해결책을 『고용』하고 『해고』하는지를 밝히는 방법론입니다. 단순히 "왜 전환했는가"를 직설적으로 묻지 않고, 『전환 사건』의 스토리를 재구성하여 Push·Pull·Habits·Anxieties 4동인을 포착합니다.**

본 프로젝트는 **"체계적 학습 여정"이라는 진전(progress)**을 제공하려 합니다. 따라서 JTBD는 본 프로젝트의 본질과 가장 잘 맞는 리서치 방법론이며, 위 5개 질문에 답할 수 있는 유일한 경로입니다.

### 1.4 인터뷰의 세 가지 목적

1. **가설 검증**: AOS × DOS 매트릭스에서 도출된 Q1 7개 Pain이 실제 사용자의 언어·상황·동기와 일치하는가?
2. **Switch 해석**: 사용자가 현재 대체재(AI·유튜브·책)를 어떤 순간에 고용했고, 어떤 순간에 해고하는가? 본 프로젝트는 그들의 "다음 해고"가 되지 않을 수 있는가?
3. **MVP 수용성**: 본 프로젝트의 핵심 기능(105편 완결·스탬프 맵·OX 체크·교안 PDF)이 그들의 Job에 얼마나 정합하는가?

## 2. 핵심 타겟 고객 페르소나 (선정 기준)

### 2.1 대상 풀 — Persona Spectrum Map 8인

『페르소나 검증 리포트』에서 평균 4.0 이상으로 선별된 8인이 대상 풀입니다. 이들은 이미 공식 통계·시장 지표와의 정합성이 검증된 합성 프로필이므로, **랜덤한 신규 인물이 아니라 이 8인의 실물 대응자**를 인터뷰합니다.

### 2.2 인터뷰 대상 선정 기준 (스크리닝 3층 필터)

JTBD 가이드 카드의 『고유한 참가자 모집』 원칙 — 『전환 행동을 이해하기 위해 신규·이탈·무인지 세 그룹을 스크리닝한다』 — 를 본 프로젝트에 맞춰 재해석합니다.

**본 프로젝트는 아직 출시 전이므로** 전통적인 세 그룹 대신 **경제 학습에서의 전환 사건 경험 유무**로 스크리닝합니다.

| 스크리닝 층위 | 질문 | 해석 |
|---|---|---|
| **1층 · 페르소나 매핑** | 해당 지원자가 8인 중 누구에 해당하는가 | 어느 페르소나의 실물 대응자인가 |
| **2층 · 전환 사건 확인** | 최근 12개월 내 경제 학습을 "시작/중단/재개"한 계기 사건이 있었는가 | Switch 스토리가 존재하는 사람만 대상 |
| **3층 · 대체재 사용 경험** | AI·유튜브·책·강의 중 2개 이상 실제 사용 경험이 있는가 | Hiring/Firing 대화 가능성 |

세 층을 모두 통과한 사람만 인터뷰 대상으로 확정합니다.

### 2.3 우선순위 — AOS × DOS 매트릭스 기반 선정

인터뷰 리소스가 유한하므로 DOS가 높은 Pain을 가진 페르소나를 우선합니다.

| 우선순위 | 페르소나 | 대응 Pain (DOS) | 인터뷰 인원 |
|---|---|---|---|
| **1순위** | 박지훈 (Q1-A 청년 각성) | P1 체계감 (DOS 3.80) | **5명** |
| **1순위** | 이수민 (Q1-A 실패 회복) | P5 깊이공백 (3.80), P4 신뢰 (2.70) | **5명** |
| **2순위** | 장은혜 (교사) | P14 학생경로 (3.40), P13 수업부담 (2.40) | **3명** |
| **2순위** | 정해민 (Q1-B) | P7 판단기준 (2.40) | **3명** |
| **3순위** | 한정숙 (Q1-C) | 설계 기준 검증 (매체 선택권) | **2명** |
| **3순위** | 오세은 (극단) | P16 세션단편 (2.10) | **2명** |
| **참고** | 김성호 (극단) | P15 저시력 (1.80) | **1명** |
| **비대상** | 서하윤 (비활성) | P18 자기규정 (DOS 2.00, 도달 불가) | **0명** |

**총 21명**. Stage 1 파일럿 이전 완료를 목표로 합니다.

### 2.4 서하윤을 제외하는 이유

서하윤(비활성)은 AOS 4.00으로 최상위이지만, 본질적으로 **경제 학습을 거부하는 상태**입니다. JTBD 가이드 카드는 『전환 행동을 이해하는 것이 핵심』이라 명시했는데, 서하윤은 전환을 한 적이 없으므로 Switch 인터뷰 대상이 되지 못합니다. 장기 미션 나침반으로만 유지하고 본 인터뷰에서는 제외합니다.

## 3. 검증을 위한 핵심 가설 (Job-Story 기반)

### 3.1 Job-Story 표기법

```
[특정 상황(Situation)]에 있을 때,
[동기(Motivation)]를 위해,
[해결하려는 과업(Expected Outcome)]을 하고 싶지만,
[장애물(Obstacle)] 때문에 어렵다.
```

### 3.2 페르소나별 Job-Story 가설

각 페르소나의 AOS/DOS 상위 Pain을 Job-Story로 변환했습니다. 인터뷰에서 이 가설이 실제 사용자의 언어와 일치하는지 검증합니다.

#### JS-1 · 박지훈 (Q1-A 청년 각성) — P1 대응

> **첫 월급을 받고 경제에 대한 무지를 자각한 27세 직장인으로서**,
> **불안과 조급함을 해소하고 『배우고 있다』는 감각을 얻기 위해**,
> **경제를 체계적으로 이해하고 싶지만**,
> **AI·유튜브·책의 파편 정보 속에서 내가 어디까지 왔는지 알 수 없어 반복적으로 포기한다.**

#### JS-2 · 이수민 (Q1-A 실패 회복) — P5·P4 대응

> **코인·주식 투자 실패 후 "기초부터 다시"라고 결심한 29세 직장인으로서**,
> **진중한 학습 여정에 자신을 다시 맡기기 위해**,
> **깊이 있되 혼자 완주 가능한 학습 경로를 찾고 싶지만**,
> **경제학원론은 너무 무겁고 유튜브는 너무 얕고 유료 강의는 후킹이 강해서 어디에도 정착할 수 없다.**

#### JS-3 · 장은혜 (교사) — P14·P13 대응

> **중학교 사회 수업에서 경제 단원을 가르쳐야 하는 36세 교사로서**,
> **수업 준비 부담을 줄이고 학생이 교실 밖에서도 스스로 탐색할 수 있게 하기 위해**,
> **영상·교안·OX·판서 자료가 하나로 정렬된 무료 수업 패키지를 쓰고 싶지만**,
> **EBS·교육청 자료·유튜브를 매번 조합하느라 수업 한 차시 준비에 2시간 이상 든다.**

#### JS-4 · 정해민 (Q1-B) — P7 대응

> **3년 내 내집마련을 계획하는 41세 직장인으로서**,
> **가족 자산 규모의 판단을 내가 직접 책임지기 위해**,
> **부동산·금리의 판단 근거를 내 언어로 이해하고 싶지만**,
> **유튜브 채널들이 상충하는 말을 하고 시간은 주말 1~2시간뿐이어서 판단 기준이 형성되지 않는다.**

#### JS-5 · 한정숙 (Q1-C) — 설계 기준 검증

> **대기업 퇴직 2년차 58세 은퇴자로서**,
> **전문가·금융기관에 휘둘리지 않고 스스로 판단하기 위해**,
> **퇴직금·연금·상속의 기본 용어부터 천천히 이해하고 싶지만**,
> **유튜브 영상은 속도가 빠르고 디지털 UI 자체가 낯설어 진입이 어렵다.**

#### JS-6 · 오세은 (극단) — P16 대응

> **14개월 아기를 양육 중인 육아휴직 33세 전 금융권 직장인으로서**,
> **복직 전까지 경제 판단력을 되살리기 위해**,
> **5~10분 단위 세션이 누적되는 학습 구조를 원하지만**,
> **긴 영상은 시작조차 어렵고 소리를 낼 수 없는 환경이어서 학습이 파편화된다.**

### 3.3 7개 공통 검증 명제

개별 Job-Story를 넘어 **8인 전체에 걸쳐 확인해야 할 공통 명제 7개**를 설정합니다. Stage 1 파일럿의 성공 기준과 직접 연결됩니다.

| 명제 | 검증 질문 | 연결 원칙/KPI |
|---|---|---|
| **M1** | 『체계감』의 부재가 반복 이탈의 주원인이다 | 원칙 5, P1, DOS 3.80 |
| **M2** | "진중한 톤"이 "후킹 톤"보다 실제 학습 지속을 낳는다 | 원칙 3, P4·P6 |
| **M3** | AI는 개별 답에서만 이겼지 여정에서는 이기지 못한다 | 5 Forces 대체재 대응 |
| **M4** | 교사의 "수업 준비 시간"이 교안 채택의 결정 변수다 | P13, 교사 모드 정당성 |
| **M5** | 매체 통합(영상·글·책)이 실제로 서로를 강화한다 | 원칙 4, P11 |
| **M6** | 짧은 세션 설계가 Q1 전체에 이익이 된다 | P16, 보편 설계 |
| **M7** | 원칙 5(부분 공개 금지)가 Q1-B에게 구조적 장벽이 된다 | 정해민 정책 결정 |

## 4. 심층 인터뷰 질문지

### 4.1 인터뷰 가이드 카드 4단 구조 준수

업로드된 가이드 카드의 4단 구조를 반영합니다.

| 가이드 단계 | 본 인터뷰 대응 |
|---|---|
| ①준비·구조 | 60~90분 / 페르소나 스크리닝 기반 모집 |
| ②심층 목표 | 직설적 "왜 전환?" 금지 / 전환 사건 스토리텔링 유도 |
| ③핵심 질문 | 대안·맥락·감정·연상 질문으로 구조화 |
| ④4가지 전환 동인 | Push·Pull·Habits·Anxieties 경청 |

### 4.2 전체 흐름 (60~90분 기준)

| 단계 | 시간 | 목적 | 분위기 |
|---|---|---|---|
| **① 오프닝·라포** | 5~10분 | 안심 + 동의 확인 | 가벼움 |
| **② 일상·현재 맥락** | 10~15분 | 학습자 상태 파악 | 자연스러운 대화 |
| **③ 전환 사건 재구성** | 20~30분 | Switch 스토리 발굴 | 감정 안전 |
| **④ 대체재 경험** | 15~20분 | Hiring/Firing 분석 | 구체 사례 요구 |
| **⑤ 본 프로젝트 반응** | 10~15분 | MVP 수용성 테스트 | 중립 |
| **⑥ 클로징** | 5분 | 감사 + 후속 연락 동의 | 따뜻함 |

### 4.3 질문지 상세

#### 🟢 Part ① 오프닝·라포 (5~10분)

**목적**: 인터뷰이가 "정답을 말해야 한다"는 부담을 내려놓게 하는 단계.

- "오늘은 정답이 있는 자리가 아닙니다. 평소 경제에 대해 어떻게 느끼시는지를 그대로 들려주시면 됩니다."
- "녹음에 동의해주시고, 녹음된 내용은 분석 목적 외에 쓰이지 않습니다."
- "중간에 답하고 싶지 않은 질문이 있으면 언제든 넘기셔도 됩니다."

#### 🟢 Part ② 일상·현재 맥락 (10~15분)

**목적**: 학습자의 현재 위치 확인. AOS/DOS 가설과의 1차 교차 검증.

**핵심 질문 (KQ)**
- KQ1. "요즘 하루를 어떻게 보내고 계신가요?" *(열린 질문)*
- KQ2. "경제에 대해 생각할 일이 언제 있으세요?"
- KQ3. "지금 경제에 대해 알아야 하거나 궁금하다고 느끼는 게 있다면 뭔가요?"

**후속 질문 (FQ)**
- FQ1. "그 상황에서 보통 어떻게 하세요?"
- FQ2. "마지막으로 그런 생각이 드셨던 건 언제였나요?" *(최근 사건 유도)*
- FQ3. "뭐가 걸림돌이었나요?"

#### 🟢 Part ③ 전환 사건 재구성 (20~30분) — **인터뷰의 핵심**

**목적**: 학습을 시작·중단·재개한 『Switch Event』의 스토리를 사용자의 언어로 재구성.

> **가이드 카드 원칙**: "단순히 왜 전환했는지 직설적으로 묻지 말고, 전환 사건에 초점을 맞추고 스토리텔링을 유도하라."

**핵심 질문 (KQ)**
- KQ4. "경제를 '제대로 배워야겠다'고 처음 느꼈던 순간이 있으신가요? 그 장면을 떠올려서 들려주세요."
- KQ5. "그날은 어떤 하루였나요? 그 전에 무슨 일이 있었어요?"
- KQ6. "그 직후에 뭘 하셨어요?"
- KQ7. "그 시도가 지금은 어떻게 됐어요?"

**연상 유도 질문 (가이드 카드 3번 카드 적용)**
- FQ4. "그때 하루 중 몇 시였나요?"
- FQ5. "혼자 계셨어요, 누구랑 함께 있었어요?"
- FQ6. "어떤 기분이셨어요? 한 단어로 표현한다면?"
- FQ7. "그 일이 있기 전과 후, 본인에 대한 생각이 달라진 게 있나요?"

**Push 동인 포착 (가이드 카드 4번 카드 적용)**
- FQ8. "그때 '이대로는 안 되겠다'고 느끼게 만든 게 뭐였어요?"
- FQ9. "만약 그 일이 없었다면 지금도 똑같이 지내고 계셨을까요?"

#### 🟢 Part ④ 대체재 경험 — Hiring/Firing 분석 (15~20분)

**목적**: AI·유튜브·책·강의 등 대체 솔루션과의 관계를 "고용/해고"의 서사로 재구성.

**핵심 질문 (KQ)**
- KQ8. "그 후에 경제를 배우려고 시도한 것들을 하나씩 이야기해 주실 수 있나요? 시간 순서로요."
- KQ9. "각각을 써보기 전에는 뭘 기대하셨어요?"
- KQ10. "써보고 나니 어땠어요? 계속 쓰시나요, 그만두셨나요?"
- KQ11. "그만두셨다면 언제, 왜 그만두셨어요? 그때 뭐 하고 계셨어요?"

**Pull 동인 포착**
- FQ10. "처음 그걸 써보려고 했던 결정적 이유는 뭐였어요?"
- FQ11. "지금 돌아보면 거기서 진짜 얻은 게 뭐였어요?"

**Habits·Anxieties 포착**
- FQ12. "새로운 걸 시도할 때 망설여지는 게 있으세요? 어떤 점이요?"
- FQ13. "경제 공부를 하면서 '내가 이걸 계속할 수 있을까' 하는 생각이 든 적 있으세요?"

**AI 특화 질문 (5 Forces 대체재 위협 직접 검증)**
- FQ14. "요즘은 ChatGPT 같은 AI로도 경제 질문을 하실 수 있잖아요. AI로 충분하다고 느끼신 적 있으세요? 반대로 AI로는 부족하다고 느낀 순간도 있으신가요?"

#### 🟢 Part ⑤ 본 프로젝트 반응 — MVP 수용성 테스트 (10~15분)

**목적**: 본 프로젝트의 핵심 컨셉을 **편향 없이** 제시하고 반응을 포착.

> **주의**: "이런 서비스가 있으면 쓰시겠어요?" 식의 직설적 수요 조사는 금지. 『가이드 카드』 원칙 — 표면 답변 회피 — 에 위배됩니다. 대신 **컨셉을 중립적 언어로 제시하고 사용자가 현재의 불만과 연결 짓는지 관찰**합니다.

**컨셉 제시문 (공통)**
> "제가 요즘 어떤 프로젝트 이야기를 듣고 있는데요. 105개 주제를 영상·교안·책으로 묶어서 무료로 제공하는 경제 입문 교과서 같은 프로젝트라고 해요. 자극 없이 차근차근 이어지는 구조라고 하더라고요. 어떻게 들리세요?"

**핵심 질문 (KQ)**
- KQ12. "솔직한 첫인상은 어떠세요?"
- KQ13. "본인에게 이게 필요할 것 같으세요, 아닐 것 같으세요? 왜 그렇게 느끼세요?"
- KQ14. "반대로, 이런 게 있다면 걱정되는 점은 뭐예요?" *(Anxiety 포착)*

**페르소나별 심화 질문** (대상에 따라 선택 적용)
- *(박지훈·이수민)* FQ15. "출퇴근 지하철 30분에 한 편씩 보는 형태인데, 영상 끝나고 OX 퀴즈가 나온다고 해요. 어떠세요?"
- *(정해민)* FQ16. "처음부터 순서대로 봐야 한다고 하네요. 부동산 주제만 먼저 보는 건 안 되고요. 어떠세요?" *(원칙 5 수용성 직접 검증 — M7 명제)*
- *(장은혜)* FQ17. "영상 한 편당 교사용 교안이 따로 나와서, 그걸 바로 수업에 쓸 수 있는 PDF로 받는 구조예요. 선생님 수업 흐름에 맞을까요?"
- *(한정숙)* FQ18. "영상 말고도 글로 된 자료와 나중엔 종이책으로도 나올 예정이라고 해요. 어떠세요?"
- *(오세은)* FQ19. "5~10분 단위로 잠깐 보고 나중에 이어서 볼 수 있다고 해요. 이게 실제로 도움이 될까요?"

#### 🟢 Part ⑥ 클로징 (5분)

- "오늘 이야기해주신 것 중에 인터뷰 외에 개인적으로 계속 고민되시는 게 있으실까요?"
- "이후 프로젝트가 진행되면 초기 테스터로 다시 연락드려도 괜찮을까요?"

### 4.4 질문에서 반드시 피해야 할 표현

가이드 카드의 『표면적 답변 피하기』 원칙:

| ❌ 금지 질문 | ⭕ 대체 질문 |
|---|---|
| "왜 전환하셨어요?" | "그때 무슨 일이 있었는지 이야기해 주세요." |
| "어떤 기능이 필요하세요?" | "마지막으로 그것을 하려고 했을 때 어떻게 하셨어요?" |

### 4.5 경청해야 할 4가지 동인 (가이드 카드 4번 카드)

인터뷰 중 다음 신호가 나오면 **즉시 FQ로 심화**합니다.

| 동인 | 경청 포인트 | 심화 FQ |
|---|---|---|
| **Push (상황의 밀어냄)** | "이대로는 안 되겠다", "화가 났다", "무섭다" | "그 순간을 조금 더 자세히 이야기해 주세요" |
| **Pull (새 해결책의 끌어당김)** | "이거 괜찮아 보이더라", "궁금했다" | "그 기대가 어디서 왔어요?" |
| **Habits (발목 잡는 습관)** | "원래 이렇게 했어요", "늘 그랬어요" | "그 습관이 이번에도 발목을 잡았나요?" |
| **Anxieties (새로운 것에 대한 불안)** | "걱정됐어요", "망설였어요" | "구체적으로 뭐가 걱정이었어요?" |

## 5. 인터뷰 수행 절차 및 방법론

### 5.1 인터뷰 대상별 전략 설계 (1단계 · 전략 초안)

『인터뷰 대상별로 인터뷰 전략을 정리한 뒤 페르소나 정합성 검토 단계를 거친다』는 요구사항에 따라, 전략을 **대상별로 먼저 설계**합니다.

| 대상 페르소나 | 인터뷰 전략 초안 |
|---|---|
| **박지훈 (5명)** | 출퇴근·월급·뉴스 트리거 중심. AI 사용 패턴 집중 추적. 체계감 부재의 실제 표현 수집. |
| **이수민 (5명)** | 투자 실패 사건의 스토리 재구성 핵심. 후킹 거부감의 정량화. 입문서 이탈 경로 재현. |
| **장은혜 (3명)** | 수업 준비 루틴 현장 재현. EBS·교육청 자료 사용 히스토리. 교안 채택 결정 변수 도출. |
| **정해민 (3명)** | 주택 의사결정 타임라인 중심. 주제별 진입 요구의 강도 측정 (M7 검증). |
| **한정숙 (2명)** | 퇴직 이벤트 후 학습 시도 경로. 디지털 UI 좌절 구간 재현. |
| **오세은 (2명)** | 하루 시간 맵핑. 5~10분 단위 세션의 실제 가능성. 소리 없는 환경의 빈도. |
| **김성호 (1명)** | 접근성 제약 구간 재현. 기존 도구(팟캐스트·오디오북)의 한계. |

### 5.2 페르소나 정합성 검토 (2단계 · 전략 검증)

각 대상의 전략이 **해당 페르소나의 핵심 속성**과 정합하는지 점검합니다.

| 페르소나 | 핵심 속성 (검증 리포트 기준) | 전략 정합성 | 판정 |
|---|---|---|---|
| 박지훈 | 디지털 네이티브 · 짧은 학습 시간 · AI 대체 위협 최고 | AI 사용 패턴 집중 추적 ✅ | ✅ 정합 |
| 이수민 | 투자 실패 트리거 · 후킹 거부감 · 진중한 톤 선호 | 실패 사건 스토리 재구성 ✅ | ✅ 정합 |
| 장은혜 | 수업 준비 부담 · 교안 채택 자유 | 수업 준비 루틴 재현 ✅ | ✅ 정합 |
| 정해민 | 주제별 진입 선호 · 의사결정 마감 존재 | 주택 타임라인 중심 ✅ + M7 검증 ✅ | ✅ 정합 |
| 한정숙 | 매체 선호 극단 · 디지털 UI 장벽 | 디지털 UI 좌절 구간 재현 ✅ | ✅ 정합 |
| 오세은 | 시간 단편화 · 소리 낼 수 없는 환경 | 시간 맵핑 + 소리 환경 검증 ✅ | ✅ 정합 |
| 김성호 | 저시력 제약 · 오디오 중심 학습 | 기존 오디오 도구 한계 탐색 ✅ | ✅ 정합 |

**결과**: 7개 페르소나 전략 모두 정합. 수정 없이 실행 가능.

| **Pain 실제 언어 맵** | AOS/DOS 18개 Pain의 실제 사용자 표현 수집 | 인터뷰 종료 +3주 |

| **검증 결과 보고서** | 7개 검증 명제(M1~M7)의 검증/반증/유보 판정 | 인터뷰 종료 +4~5주 |

### 6.2 분석 방법론

#### 6.2.1 Job Map 작성
각 페르소나의 Job을 **8단계 Job Map** (Define → Locate → Prepare → Confirm → Execute → Monitor → Modify → Conclude)으로 정리합니다. 병목 단계가 MVP 기능 설계의 우선순위가 됩니다.

#### 6.2.2 Outcome-Driven Innovation (ODI)
인터뷰에서 수집된 Desired Outcomes를 Importance·Satisfaction 리커트로 재측정합니다. 기존 AOS·DOS 매트릭스의 합성 추정치를 **실측치로 교체**하고, MVP 기능 우선순위를 재확정합니다.

#### 6.2.3 Switch Canvas
Christensen의 Switch Framework에 맞춰 각 페르소나의 Push·Pull·Habits·Anxieties 4동인을 1페이지 캔버스로 정리합니다. 마케팅 메시지의 핵심 앵커가 됩니다.

#### 6.2.4 명제 검증표
7개 명제(M1~M7)에 대해 ✅ 검증 / ❌ 반증 / 🟡 유보 판정을 내립니다.

### 6.4 Stage 1 파일럿 Exit 기준 보강안

AOS × DOS 매트릭스에서 제안한 Exit 기준을 인터뷰 결과로 다음과 같이 보강합니다.

```
Stage 1 파일럿 Exit 기준 (최종)

1. DOS ≥ 3.0 Pain (P1·P5·P14) 중 2개 이상에서 체감 변화 응답률 60% 이상
   → 인터뷰에서 '체감 변화'의 조작적 정의 확보됨
   
2. DOS 2.0~2.9 Pain (P4·P7·P13·P16) 중 1개 이상에서 긍정 신호
   → 인터뷰에서 '긍정 신호'의 실제 언어 확보됨

3. 명제 M1·M3 검증 성공
   → 체계감 부재가 이탈 주원인 / AI가 여정에서는 못 이김

4. 정해민(Q1-B) 정책 결정 완료
   → 명제 M7 검증 결과에 따라 원칙 5 수용 범위 확정

5. 접근성 체크리스트(김성호·한정숙 기준) 충족
```

## 부록 C. 이 계획서의 한계

1. **21명 인터뷰는 통계적 대표성이 아닌 질적 깊이를 위한 표본**. 정량 일반화에는 부적합.
2. **서하윤(비활성)은 대상에서 제외**. 장기적으로는 별도 리서치 방법(예: 민속지학·일기 연구)이 필요.
3. **모집 채널 편향 가능성** — 특정 커뮤니티·지역에 편중되지 않게 채널 다양화 필요.
4. **인터뷰어 편향** — 창업가 단독 진행이므로 확증 편향 위험. 첫 3건은 내부 지인과 파일럿 롤플레잉 필수.
5. **본 계획서는 시작 전 v1.0**. 첫 3~5건 인터뷰 후 질문지·모집 전략·분석 프레임을 재검토해야 함.
